Step 1: Add Missing Application Images
───────────────────────────────────────

Option A: If you have the source code with Dockerfiles
  cd /path/to/catool
  podman build -t 132.186.17.22:5000/catool:1-0-0-beta .
  podman push 132.186.17.22:5000/catool:1-0-0-beta --tls-verify=false
  
  cd /path/to/catool-ns
  podman build -t 132.186.17.22:5000/catool-ns:67-g0a02ff6 .
  podman push 132.186.17.22:5000/catool-ns:67-g0a02ff6 --tls-verify=false
  
  cd /path/to/catool-ui
  podman build -t 132.186.17.22:5000/catool-ui:259-g0719cf3 .
  podman push 132.186.17.22:5000/catool-ui:259-g0719cf3 --tls-verify=false

Option B: If you can access the original registry
  podman pull perfteam-registry.advantest.com/catool:1-0-0-beta
  podman tag perfteam-registry.advantest.com/catool:1-0-0-beta \
              132.186.17.22:5000/catool:1-0-0-beta
  podman push 132.186.17.22:5000/catool:1-0-0-beta --tls-verify=false
  
  # Repeat for catool-ns and catool-ui

After pushing, refresh the web UI to see the new images!

Step 2: Configure Kubernetes Nodes
───────────────────────────────────

Run this on EACH Kubernetes node:
  bash /root/Downloads/configure-k8s-insecure-registry.sh

Or manually add to /etc/containerd/config.toml:
  [plugins."io.containerd.grpc.v1.cri".registry.configs."132.186.17.22:5000"]
    [plugins."io.containerd.grpc.v1.cri".registry.configs."132.186.17.22:5000".tls]
      insecure_skip_verify = true
  
  [plugins."io.containerd.grpc.v1.cri".registry.mirrors."132.186.17.22:5000"]
    endpoint = ["http://132.186.17.22:5000"]

Then: systemctl restart containerd

Step 3: Restart Deployments
───────────────────────────

Once images are in registry and nodes are configured:
  kubectl rollout restart deployment -n catool
  kubectl rollout restart deployment -n catool-ns
  kubectl rollout restart deployment -n catool-ui

Monitor:
  kubectl get pods -n catool -w

❓ Web UI shows no images?
   → Check registry is running: podman ps | grep registry
   → Test API: curl http://132.186.17.22:5000/v2/_catalog

❓ Can't access from browser?
   → Verify port is open: ss -tuln | grep 8080
   → Try SSH tunnel: ssh -L 8080:132.186.17.22:8080 root@132.186.17.22
   → Then access: http://localhost:8080

❓ Pods stuck in ImagePullBackOff?
   → Images might not be in registry yet
   → Check registry UI to confirm images exist
   → Verify K8s nodes configured for insecure registry
   → Check: kubectl describe pod <pod-name> -n <namespace>

❓ Want to delete an image?
   → Use the web UI - click on image tag, then delete button
   → Or via API: curl -X DELETE http://132.186.17.22:5000/v2/<name>/manifests/<tag>
